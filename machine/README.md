# LMS Project

## Документация

Интерактивный тренажер для отработки практических действий персонала

### Важно
- Тестирование добавлять не надо
- Ветки называются по принципу ГОД_НОМЕР_dev (2025_12_dev).
- Во всех тренажерах 2025г, клик идет не на квадратики рядом, а на сами элементы схемы. По дефолту должны быть видны только в ?admin режиме.

#### Схема (exercise)

- Масштабирование, перетаскивание, раскрытие на полный экран
- Масшатибрование:
  - нажатием на кнопки с иконкой лупы +/- "Приблизить" и "Отдалить" в левом краю рабочей области.
  - прокруткой колесика мыши
  - с помощью клавиатуры (кнопки +/-)
- Перемещение по схеме (если применено "+" масштабирование):
  - перемещение мыши с зажатием ЛКМ в рабочей области.
  - стрелки на клавиатуре (стрелки вверх/вниз/влево/вправо)
- Сброс схемы в изначальный масштаб при помощи кнопки с иконкой перечеркнутой лупы - "Сбросить зум" или кнопка R на клавиатуре
- Раскрытие на полный экран
  - кнопкой с иконкой "На весь экран" или кнопкой F на клавиатуре
  - для выхода из полного экрана нажатием на кнопку с иконкой "Свернуть" или кнопка Esc/F на клавиатуре
  - навигация (перетаскивание, масштабирование) при полном экране работает аналогично как при нормальном режиме
- Масштаб и положение схемы сохраняются при перезагрузке страницы
- Индикация состояния элементов
  - цвет
  - поворот
  - размер
  - текст
  - уровень заполнения
- Взаимодействие с элементами
  - обработка нажатия на элемент
  - эффекты взаимодействия: кнопка или bounce (подробнее в разделе 4. Триггеры - effect)
  - хинт (тултип) с названием элемента при наведении по умолчанию.<br>
    Может быть изменён на статус элемента (раздел 5. Индикаторы - status)<br>
  - Модальное окно выбора действия<br>
    - Открывается при взаимодействии с элементом.<br>
    - Используется для выбора или подтверждения действия.<br>
    - Закрытие по нажатию (х) не влияет на алгоритм.<br>
    - Для простых действий, (например двухпозиционному переключению) скрывается в настройках триггера. (4. Триггеры - immediate)<br>
      <br>
       **Типы модальных окон**:
      - Окно тестирования - Заголовок: "Вопрос № \_", Подзаголовок: "Текст вопроса"
      - Выбор - Заголовок: "Выберите действие", Подзаголовок: "Название действия"
  - уведомление<br>
    - уведомления появляется после выполнения определённых действий в правом верхнем углу рабочей области.
    - Не влияет на алгоритм.
    - можно закрыть вручную (х).
    - Текст задаётся разработчиком (8. Машина состояний - notify).
- Алгоритм
  - Состоит из шагов, каждый шаг описывает конкретное состояние элементов системы
  - Переход на следующий шаг осуществляется при совершении верного действия на правильном элементе в контексте текущего шага.
  - Типы переходов между шагами:
    - **Последовательный** - шаги, которые можно выполнить только в заранее определенном порядке.
    - **Параллельный** - несколько шагов, которые можно выполнить в любом порядке.<br>
      Варианты шагов в группе:
      - все шаги обязательны к выполнению
      - один или несколько из шагов обязательны к выполнению
    - **Опциональный** - необязательный шаг, с текущего шага можно как перейти на него, так и перейти на другой по алгоритму, не выполняя этот.
  - При завершении алгоритма отображается модальное окно "Задание завершено. Вы набрали \_ из 100 баллов"
  - При завершении алгоритма схема становится недоступной для взаимодействия
  - При перезагрузке сохраняется текущий шаг алгоритма и состояние элементов системы.

#### История (history)

- Начисление баллов
  - правильное действие не изменяет баллы,
  - неправильное действие убавляет набранные баллы на -1.
  - количество списываемых за ошибку баллов можно настроить вручную при необходимости:
    - в разделе score файла [configs/settings.ts](configs/settings.ts) можно изменить штрафы глобально.
    - можно добавить кастомный штраф вручную, при определенном действии (8. Машина состояний - Алгоритм заполнения - penalty)
- Неправильные действия:
  - неправильный ответ на тестовый вопрос
  - попытка взаимодействия с неподходящим элементом
  - выбор неверного действия на правильном элементе
  - пропуск опционального шага
- Информация при взаимодействии с элементами схемы
  - при выборе правильного элемента и выполнении правильного действия на нем в Истории записывается "Название элемента" и "Выполненное действие"
  - при выборе неправильного элемента: "Название элемента" и "Выбрано неверно"
  - при выполнении неправильного действия после правильного выбора элемента: "Название элемента", "Выполненное действие" и "-1" балл.
  - при завершении всех шагов в группе: "Этап завершен" "Название этапа"
- Информации при прохождении тестирования
  - отображается "Вопрос теста" и "Ответ пользователя"
  - при окончании тестирования выводится сообщение "Тест завершен. Правильных ответов _ из _".
- Итоговые баллы
  - отображается итоговое количество набранных баллов под историей действий.
  - в конце прохождения упражнения вычитаются баллы за пропущенные опциональные шаги в виде "Дополнительный шаг пропущен" "Название шага"
  - изначально 100 баллов, из которых вычитаются баллы за неправильные действия.
- При перезагрузке страницы история сохраняется
- Для корректной работы истории необходимо, чтобы каждому действию соответствовал свой уникальный шаг (step). Важно для групп и параллельных шагов.

#### Тестирование (quiz) - В текущей версии не используется

- Тестирование начинается с нажатия на кнопку "Тестирование"
- Без прохождения тестирования взаимодействие со схемой недоступно<br>
- После прохождения тестирования кнопка "Тестирование" блокируется
- На каждый вопрос теста можно дать только один ответ
- Неправильный ответ уменьшает набранные баллы на -1
- Ответы каждый раз выводятся в случайном порядке
- При перезагрузке страницы сохраняется: текущий вопрос, порядок ответов, выбранный ответ

#### Шаги (step)

- Перечень шагов для быстрой навигации администратора по алгоритму
- Отображаются только для администратора
- Соответствуют шагам, прописанным в алгоритме (Excel). 
- Если одному шагу из Excel соответствует группа параллельных шагов - для подшагов рекомендуется использовать запись типа:<br>
  <номер шага из Excel>.<Номер в группе>. (Например 1.1, 1.2 ...)
- Шаги и схема синхронизированы между собой
  - при взаимодействии с элементом на схеме, отмечается соответствующий шаг в списке
  - при выборе шага в списке, выполняются все предыдущие шаги (в т.ч. текущий).<br>
    Выполнение шагов сопровождается с соответствующей шагам индикацией элементов на схеме, вызовом уведомлений, записью в Историю.
- При наведении на шаг отображается хинт (тултип) с заголовком шага (7. Шаги алгоритма - title)
- По шагам можно перемещаться в любом направлении
- Если есть тестирование, то оно должно идти первым шагом
- Если есть опциональный шаг, то он не должен стоять после шагов, параллельных ему
- При перезагрузке страницы список шагов сохраняется в том же состоянии.

#### Режим Админа

- Включает возможность для навигации между шагами и отображает вспомогательные числа у элементов на схеме
- Для перехода добавить в конец адресной строки `?admin`
- Не в Админ режиме навигация и вспомогательные числа скрываются

#### 1. Схема

- Схема - SVG элемент, отображаемый на странице, визуализирующий техническую установку
- Основная и дополнительные схемы добавляются в папку: [configs/schemes](configs/schemes)
- Далее заполняется [configs/schemes/index.ts](configs/schemes/index.ts)
<br>
`SCHEMES_CONFIG` представляет из себя объект, содержащий схемы тренажера.<br>
В качестве ключа объекта используется id. схемы. <br>
Впоследствии id схемы будет использоваться для настройки триггеров и индикаторов.<br><br>
Каждому id схемы соответствует объект настроек вида:<br>
1. `type` - тип схемы
   - 'main' - для основой схемы, появляется в рабочей области.
   - 'modal' -  дополнительная схема, появляющаяся в модальном окне
2. `component` - импортируемый SVG элемент
3. `title` - только для type === 'modal'. Отображается как заголовок модального окна
4. `size` - только для type === 'modal'. Определяет размер модального окна
  
Примеры:
```[json]
// Пример основной схемы

  'mainScheme': {
    type: 'main',
    component: MainScheme,
  },

// Пример дополнительной схемы

  'additionalScheme1': {
    type: 'modal',
    title: 'Панель управления',
    size: 800,
    component: ModalScheme,
  },
```

<br><br>
Схема не появится автоматически в рабочей области, если следующие конфиги не заполнены верно и непротиворечиво:

! Если схема не отображается - попробуйте очистить localStorage браузера.
! Если проблема сохраняется, убедитесь, что отсутствуют фактические ошибки в конфигах, а также ошибки типизации.<br><br>
! Если ваш тренажер является форком тренажера, где использоался quiz (например ветки engine), то схема также может не отобразиться сразу. 
Перейдите в режим администратора (?admin) и переключите шаг с первого на другой вручную. Схема должна появиться. 
Повторно это действие совершать не потребуется<br><br>
***

**Требования к SVG элементам (при несоответствии - обратиться к дизайнеру за правками):**

  - если к элементу будет применен `range` (с изменением размера/шкала), то изначально он должен быть заполнен на 100%
  - если к элементу будет применен `scale`, то все части элемента должны быть объединены в одну группу
  - если к элементу будет применен `rotate`, то все части элемента должны быть объединены в одну группу
  - если есть какие-то дополнительные кнопки, то размещать их на svg

**Алгоритм экспорта схемы из Figma**

  1. Переходим в Figma по ссылке из [Excel](https://docs.google.com/spreadsheets/d/1nE6Hm-kQ0lmBc5BzwwZvYF42Utc1OzVXt1LRRRGqqkU/edit?gid=0#gid=0)
  2. Выделяем корневой фрейм
  3. В экспорте выбираем SVG
  4. В расширенных параметрах экспорта (...) включаем только указанные галочки:
     - Ignore overlapping layers
     - Include bounding box
     - Include "id" attribute
     - Simplify stroke
  5. Нажимаем кнопку 'Export <название фрейма>'

#### 4. Триггеры

- Список триггеров - все элементы на схеме, с которыми может взаимодействовать пользователь
- Пользователь может взаимодействовать только с элементами схемы, соответствующие реальным объектам (арматуры, насосы и т.д.)<br>
  Копки с числами - только для удобства тестирования. Пользователь с ними взаимодействовать не будет, так как они отображаются только в admin режиме.
- Для тренажеров 2024 года может быть необходимо показать дополнительные кнопки и индикаторы, которые по умолчанию скрываются.
  - Чтобы это сделать нужно перейти в файл [register-hints.ts](src/widgets/exercise-scheme/lib/register-hints.ts)
  - Изменить значение `NUMBER_PREFIX` и/или `INDICATOR_PREFIX`. Например `'button_'` => `c_button_`
- Изменения добавляются в файл: [configs/triggers.ts](configs/triggers.ts)
- Документация:
  - [интерфейс триггера](./src/entities/exercise/types/scheme-trigger.ts)
- **Алгоритм заполнения**
  1. `id` - id элемента из svg
  2. `scheme` - id схемы, в которой находится элемент
  3. `name` - название элемента из Excel, отображается в тултипе и истории
  4. `actions` - список доступных действий на элементе
     - `value` - уникальный идентификатор действия. Из него создается событие `<id триггера>.<value действия>`
     - `label` - текст действия
  5. `effect` - эффект взаимодействия с элементом
     - `button` - эффект взаимодействия как кнопка, без hover эффекта
     - `input` - добавляет возможность вставить поле для ввода значений с клавиатуры при нажатии
     - `sidebar-button` - Кнопка появляется в левом боковом меню, а не на схеме. Принцип действия аналогичен другим триггерам
  6. `immediate` - Опциональное свойство, <boolean>. 
     - true - скрывает окно выбора действия при обработке триггера. Действие выполняется сразу по нажатию. 
        Используется для двухпозиционных переключателей, кнопок с одной функцией.
     - false - стандартное поведение с окном.

#### 5. Индикаторы

- Список индикаторов - все элементы со схемы, которые изменяют свое визуальное состояние или статус в хинте
- Изменения добавляются в файл: [configs/indicators.ts](configs/indicators.ts)
- Документация:
  - [интерфейс индикатора](./src/entities/exercise/types/scheme-indicator.ts)
- **Алгоритм заполнения**
  1. `id` - id элемента из svg
  2. `scheme` - id схемы, в которой находится элемент
  3. `params` - параметры для изменения состояния элемента схемы
     - `text` - изменение текста
     - `color` - изменение цвета элемента
     - `range` - изменение уровня элемента. Позволяет изменять значение шкалы или цвет.
     - `rotate` - вращение элемента вокруг определенной точки
     - `scale` - масштабирование элемента по определенной оси<br>
       Обязательно заполнить атрибуты, которые планируется изменять.
       - `status` - задает хинт элементу. Применяется для триггеров, чтобы вместо названия триггера отображалось его состояние 
       (требование заказчика).

#### 6. Состояние схемы

- Изначальное состояние всех элементов схемы, которые будут изменять свое визуальное состояние в процессе выполнения алгоритма.<br>
  Используется при первоначальной загрузке схемы и для возврата элементов схемы в начальное состояние, после выбора "Начать заново"
- Изменения добавляются в файл: [configs/state.ts](configs/state.ts)
- **Алгоритм заполнения**
  1. ключи первого уровня - id, указанные в файле [configs/tasks.ts](configs/tasks.ts)
  2. ключи второго уровня - id, указанные в файле [configs/indicators.ts](configs/indicators.ts)
  3. значения - начальное состояние атрибута для элемента (указывается только для тех атрибутов, для которых настроены параметры)

#### 7. Шаги алгоритма

- Список шагов алгоритма в соответствии с файлом Excel. Используется для навигации по шагам (для админа)
- Изменения добавляются в файл: [configs/steps.ts](configs/steps.ts)
- Документация:
  - [интерфейс шага](src/entities/progress/types/progress-step.ts)
- **Алгоритм заполнения**
  1. ключи - id, указанные в файле [configs/tasks.ts](configs/tasks.ts)
  2. `id` - номер фактического шага в Excel<br>
  _Если одному шагу из Excel соответствует группа параллельных шагов - для подшагов рекомендуется использовать запись типа:<br>
      <номер шага из Excel>.<Номер в группе>. (Например 1.1, 1.2 ...)_
  3. `quiz` - ставим `true` если шаг является тестированием.<br>
     _Может быть только первым шагом с `id: 1`._ <br>
     Если `quiz: true`, то поля ниже (3-5) не заполнять.
  4. `title` - наименование шага, отображается в тултипе
  5. `events` - список событий для машины для того чтобы пройти текущий шаг
  6. `optional` - ставим `true` если шаг опциональный
  7. `remoteHint` - ставим `true`если хотим, чтобы хинт элемента был скрыт
  8. `skipHistory` - ставим `true`если хотим, чтобы действия этого шага не записывались в историю

#### 8. Машина состояний

- Описание состояний и переходов
- Изменения добавляются в файл: [configs/machine.ts](configs/machine.ts)
- Документация:
  - [официальная документация xstate](https://stately.ai/docs)
  - [действия машины при переходах между состояниями](src/entities/exercise/utils/machine-actions.ts)
- Демонстрация различных кейсов: [LMS: Демонстрация](https://stately.ai/registry/editor/8253157d-3bc4-4d7a-b8d9-991bc265a75d)
- Реальные схемы: [LMS: Схемы](https://stately.ai/registry/editor/b63a5f89-4780-4b3b-affa-a02971169947)
- **Алгоритм заполнения**<br>
  1. Ключи первого уровня - id, указанные в файле [configs/tasks.ts](configs/tasks.ts)
     Описать машину состояния можно в [редакторе](https://stately.ai/registry/editor),
     в таком случае пропускаются шаги 1-3, код из редактора можно скопировать в машину.
  2. Указать состояние, его название на русском
  3. Указать ожидаемое событие для состояния (в поле `on`). Шаблон события: `<id триггера>.<value действия на триггере>`.<br>
     **Системные события**
     - `stop` - при использовании `delay` необходимо обрабатывать преждевременный выход из состояния таймера по указанному событию
     - `close` - при открытии схемы в модальном окне необходимо обрабатывать преждевременное закрытие модального окна по указанному событию
  4. В переходе указать целевое состояние (`target`)
  5. Если необходимо ограничение на переход, указать `guard`<br>
     **Виды guard**
     1. `check` - проверяет значения атрибутов по указанному индикатору<br>
        Пример:
     ```ts
     guard: check('Union', s => s.range + 10 === 80)
     ```
     2. `inputMatch` - проверяет значения атрибутов по указанному диапазону значений при вводе значений с клавиатуры<br>
     Пример:
     ```ts
     guard: {type: 'inputMatch', {params: {expected: [0, 100]}}}
     
     guard: {type: 'inputMatch', {params: {expected: [40]}}}
     ```
  6. Указать действие(я) при переходе (`actions`). _Действия указываются в порядке приведенном ниже_<br>
     **Виды actions**<br> 
     1.`scheme` - заменяет текущую схему по указанному id схемы
     Примеры:
     ```ts
     actions: scheme('ГТУ 8')
     ```
     2.`change` - изменяет значения атрибутов по указанному индикатору. Соответствует id указанному в [indicators.ts](configs/indicators.ts)<br>
     Примеры:
     ```ts
     actions: change('Union', { text: 'new text' })
     ```
     ```ts
     actions: change('Union', { color: 'red' })
     ```
     ```ts
     actions: change('Union', s => ({ range: s.range + 10 }))
     ```
     3.`stage` - создает событие в истории о завершении этапа. Указывать название этапа из Excel<br>
     Пример:
     ```ts
     actions: stage('Действия до прогрева турбины')
     ```
     4.`step` - регистрирует выполнение указанного шага. _Указывать id шага из [steps.ts](configs/steps.ts)_<br>
     Пример:
     ```ts
     actions: step(2)
     ```
     5.`notify` - создает уведомление. Указывать текст уведомления из Excel<br>
     Пример:
     ```ts
     actions: notify('Перед прогревом турбины необходимо проверить подключение охлаждающей воды')
     ```
     6.`status` - позволяет задать сообщение которые должно вывестись в хинт
     Пример:
     ```ts
     actions: status('button_oil', 'Новый статус')
     ```
     7.`copy` - копирует значение из атрибута по указанному индикатору в другой индикатор
     Пример:
     ```ts
      copy('Rectangle 12', 'Rectangle 13')
     ```
     8.`penalty` - позволяет выставить количество отнимаемых баллов за ошибку в ручную
     Пример:
     ```ts
      penalty('Naming', 'Message', -10)
     ```
     9.`history` - позволяет записать кастомную историю
     Пример:
     ```ts
       history('Заголовок(обычно название арматуры', {details: 'Произведенное действие'})
     ```

  2. Вкладка Приложение,
  3. Хранилище -> Локальное хранилище -> адрес сайта

- Анимация будет автоматически выключена, если не обнаружено аппаратное ускорение
- Если нужно выключить анимацию вручную, нужно добавить в конец адресной строки `?no-animate`
